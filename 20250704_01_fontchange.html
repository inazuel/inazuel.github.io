pelican blog 폰트 변경하기

Date Fri 04 July 2025 By inazuel Category Pelican Tags python / pelican

  블로그 폰트를 변경했다. 폰트명은 D2Coding-Ver1.3.2-20180524-all으로 폰트명에서 알 수 있듯이 D2Coding, 즉 코딩용 폰트이며 네이버에서 만든 것이다. 멋은 없지만 보는 사람들이 오해하거나 잘못 읽힐 여지가 없는 폰트이다. 폰트의 변경 방법은 오로지 구글의 AI인 Gemini의 도움을 받았다.

  다운로드는 d2codingfont에서 할 수 있다.
다운로드한 폰트를 inazuel.github.io/content/static/fonts 폴더 안에 복사한다.(환경이 저와 같다면 폴더는 없을 겁니다. 이럴때는 폴더를 만들어주면 됩니다.)
pelicanconf.py ~~(또는 publishconf.py)~~에 아래의 코드를 입력한다. (저는 pelicanconf.py 기준으로 작성했습니다.)

# 기존 STATIC_PATHS에 'static' 추가
STATIC_PATHS = ['images', 'static'] # 'static'이 이미 있다면 추가하지 않아도 됩니다.

# 폰트 파일 경로 지정
EXTRA_PATH_METADATA = {
    'static/fonts/D2Coding-Ver1.3.2-20180524-all.ttf': {'path': 'static/fonts/D2Coding-Ver1.3.2-20180524-all.ttf'},
}


  content/static/css/custom.css 파일을 생성한다.
  생성된 파일에 아래의 코드를 입력한다.

/* 웹 폰트 정의 */
@font-face {
    font-family: 'D2Coding';
    src: url('/static/fonts/D2Coding-Ver1.3.2-20180524-all.ttf') format('truetype');
    font-weight: normal; /* 단일 파일이므로 normal만 정의 */
    font-style: normal;
}

/* 블로그 전체에 D2Coding 폰트 적용 */
body, h1, h2, h3, h4, h5, h6,
.navbar, .btn, .form-control,
p, li, a, span {
    font-family: 'D2Coding', monospace !important;
}

/* 코드 블록에 더 명확하게 적용 (프로그래밍 폰트이므로) */
code, pre, .highlight {
    font-family: 'D2Coding', monospace !important;
    /* 필요하다면 코드 블록의 폰트 크기 조절 */
    /* font-size: 0.9em; */
}


  다시 텍스트 편집기로 pelicanconf.py 에 아래의 코드를 추가한다.

# 사용자 정의 CSS 파일 경로
CUSTOM_CSS = 'static/css/custom.css'

빌드 및 배포
''' git add . git commit -m "Apply D2Coding font to blog using all-in-one file" git push origin master # 또는 main '''
  이것으로 폰트 변경을 완료되었다.
  아래의 접기/펼치기는 gemini의 답변을 기록용으로 남겨놨다.

접기/펼치기 ## 접은 제목 ----- D2Coding 폰트 파일명이 **`D2Coding-Ver1.3.2-20180524-all`** 인 경우를 기준으로 블로그 폰트를 변경하는 방법을 다시 자세히 알려드릴게요. 이 파일은 모든 폰트 웨이트(regular, bold 등)를 포함하는 단일 파일일 가능성이 높습니다. ----- ## D2Coding 폰트 변경 단계별 가이드 (파일명: `D2Coding-Ver1.3.2-20180524-all`) #### 1단계: D2Coding 폰트 파일 다운로드 및 준비 1. **D2Coding 폰트 다운로드**: * 네이버 D2Coding 폰트 GitHub 저장소 ([https://github.com/naver/d2codingfont](https://github.com/naver/d2codingfont))에 접속합니다. * 'Code' 버튼을 클릭한 후 'Download ZIP'을 선택하여 폰트 압축 파일을 다운로드합니다. * 압축을 해제하면, **`D2Coding-Ver1.3.2-20180524-all.ttf`** 파일을 찾을 수 있습니다. #### 2단계: 블로그 프로젝트에 폰트 파일 추가 다운로드한 폰트 파일을 블로그 프로젝트에 복사합니다. 1. **폰트 폴더 생성**: 여러분의 블로그 프로젝트 루트 디렉토리(예: `inazuel.github.io`) 안에 `content/static/fonts`와 같은 경로로 폴더를 만듭니다. ``` your-blog-repo/ ├── content/ │ ├── articles/ │ ├── pages/ │ └── static/ # 이 폴더를 생성 │ └── fonts/ # 이 폴더를 생성 ├── pelicanconf.py └── ... ``` 2. **폰트 파일 복사**: 다운로드한 **`D2Coding-Ver1.3.2-20180524-all.ttf`** 파일을 위에서 만든 `content/static/fonts` 폴더 안에 복사합니다. #### 3단계: Pelican 설정 파일 (`pelicanconf.py` 또는 `publishconf.py`) 수정 Pelican이 폰트 파일을 빌드된 사이트로 복사하도록 설정해야 합니다. 1. **`pelicanconf.py` 또는 `publishconf.py` 열기**: 텍스트 편집기로 `pelicanconf.py` (또는 `publishconf.py`를 권장) 파일을 엽니다. 2. **`STATIC_PATHS`와 `EXTRA_PATH_METADATA` 설정 추가**: `STATIC_PATHS`에 `static` 폴더를 추가하고, `EXTRA_PATH_METADATA`를 사용하여 폰트 파일이 빌드 시 `output/static/fonts/` 경로로 복사되도록 설정합니다. ```python # 기존 STATIC_PATHS에 'static' 추가 STATIC_PATHS = ['images', 'static'] # 'static'이 이미 있다면 추가하지 않아도 됩니다. # 폰트 파일 경로 지정 EXTRA_PATH_METADATA = { 'static/fonts/D2Coding-Ver1.3.2-20180524-all.ttf': {'path': 'static/fonts/D2Coding-Ver1.3.2-20180524-all.ttf'}, } ``` **주의**: `EXTRA_PATH_METADATA`의 첫 번째 경로는 `content` 폴더를 기준으로 한 상대 경로이고, 두 번째 `path`는 `output` 폴더를 기준으로 한 상대 경로입니다. #### 4단계: 사용자 정의 CSS 파일 생성 및 폰트 적용 이제 폰트 파일을 웹 페이지에서 사용할 수 있도록 CSS를 작성하고 적용합니다. 1. **사용자 정의 CSS 파일 생성**: `content/static/css/custom.css` 파일을 생성합니다. ``` your-blog-repo/ ├── content/ │ ├── static/ │ │ ├── css/ # 이 폴더 안에 │ │ │ └── custom.css # 이 파일을 생성 │ │ └── fonts/ │ │ └── ... ├── ... ``` 2. **`custom.css` 파일 내용 작성**: `custom.css` 파일에 다음 CSS 코드를 추가합니다. ```css /* 웹 폰트 정의 */ @font-face { font-family: 'D2Coding'; src: url('/static/fonts/D2Coding-Ver1.3.2-20180524-all.ttf') format('truetype'); font-weight: normal; /* 단일 파일이므로 normal만 정의 */ font-style: normal; } /* 블로그 전체에 D2Coding 폰트 적용 */ body, h1, h2, h3, h4, h5, h6, .navbar, .btn, .form-control, p, li, a, span { font-family: 'D2Coding', monospace !important; } /* 코드 블록에 더 명확하게 적용 (프로그래밍 폰트이므로) */ code, pre, .highlight { font-family: 'D2Coding', monospace !important; /* 필요하다면 코드 블록의 폰트 크기 조절 */ /* font-size: 0.9em; */ } ``` **주의**: `url()` 내의 경로는 `output` 폴더를 기준으로 한 상대 경로여야 합니다. (예: `/static/fonts/D2Coding-Ver1.3.2-20180524-all.ttf`) #### 5단계: Pelican 설정 파일에 사용자 정의 CSS 연결 `pelican-bootstrap3` 테마가 `custom.css` 파일을 블로그에 로드하도록 설정합니다. 1. **`pelicanconf.py` 또는 `publishconf.py` 열기**: 다시 텍스트 편집기로 `pelicanconf.py` (또는 `publishconf.py`) 파일을 엽니다. 2. **`CUSTOM_CSS` 설정 추가**: 다음 줄을 파일에 추가합니다. ```python # 사용자 정의 CSS 파일 경로 CUSTOM_CSS = 'static/css/custom.css' ``` #### 6단계: 빌드 및 배포 모든 파일을 수정하고 저장했으면, 변경 사항을 GitHub에 푸시하여 블로그에 반영합니다. 1. **Git 커밋 및 푸시**: ```bash git add . # 모든 변경된 파일 (폰트 파일, .py 설정 파일, .css 파일) 추가 git commit -m "Apply D2Coding font to blog using all-in-one file" git push origin master # 또는 main ``` 2. **GitHub Actions 실행 확인**: GitHub 저장소의 `Actions` 탭에서 워크플로우가 성공적으로 실행되는지 확인합니다. 3. **블로그 확인**: 배포가 완료된 후, 블로그 URL에 접속하여 폰트가 'D2Coding'으로 변경되었는지 확인합니다. **브라우저 캐시를 꼭 비워주세요 (`Ctrl + F5`)\!** ----- 이 지침대로 따라하면 D2Coding 폰트가 여러분의 블로그에 성공적으로 적용될 겁니다. 혹시 적용에 문제가 있다면, GitHub Actions 로그나 브라우저 개발자 도구(F12)의 'Network' 탭에서 폰트 파일이 제대로 로드되는지 확인해보세요.
